================================================================================
  ORBIT-TYPE DEFAULT PARAMETERS — SOURCES AND JUSTIFICATION
  for on_satellite_type_changed() in main_advanced.py
================================================================================

Document created: 2026
Purpose: Verify and document all default parameter values used when switching
         between GEO, LEO, MEO, and HEO orbit types in the satellite link
         budget simulator.

NOTE: Not all satellite link budget parameters originate from ITU-R
Recommendations. ITU-R primarily governs frequency allocations and provides
link-budget methodology. Specific satellite hardware parameters (TX power,
antenna gain, noise figure) come from system-level specifications, FCC filings,
GPS Interface Control Documents, and satellite communications textbooks.

  3.  MEO — MEDIUM EARTH ORBIT
================================================================================

Representative system: Navigation Satellite (GPS-like)
(e.g., GPS Block IIA/IIF, Galileo, GLONASS, BeiDou)

NOTE: The parameters below model a navigation satellite system. MEO can also
host communications satellites (e.g., O3b mPOWER at 8,062 km, Ka-band,
EIRP ~52 dBW). The SATELLITE_TYPES dict in constants.py uses parameters
more representative of a generic MEO (including comm satellites), hence
some values differ from the GPS-specific defaults here.

─────────────────────────────────────────────────────────────────────────────

                                          Semi-major axis a = 26 560 km,

================================================================================
  5.  KNOWN LIMITATIONS AND APPROXIMATIONS
================================================================================

1. MEO TRANSPONDER PRESET: The C-band_standard preset (6/4 GHz) is used for MEO
   because no L-band (1.6/1.575 GHz) preset exists in the application.  The
   spinner-set frequencies override the preset frequencies in the link budget
   calculation, so the final results use L-band propagation correctly.

2. MEO TX POWER: The value of 8 dBW is a compromise.  GPS Block IIF total L1
   power is approximately 15–18 dBW, but the C/A code signal alone is lower
   (~5–8 dBW for earlier blocks).  The transponder chain model recalculates
   the actual EIRP independently, so the displayed spinner value is overridden.

3. LEO ANTENNA GAIN: The 32 dBi value is specific to Starlink-class broadband
   LEO with phased arrays.  Other LEO systems (IoT, MSS) may have antenna
   gains as low as 6–25 dBi depending on frequency band and mission.

4. HEO FREQUENCY: Original Molniya satellites (1965–2000s) used VHF/UHF.
   Sirius XM uses S-band (2.32 GHz).  The Ku-band defaults represent a
   modern HEO communications system, not a specific historical system.

5. BANDWIDTH VALUES: The 240 MHz LEO bandwidth is representative of wideband
   broadband LEO, not a specific Starlink channel bandwidth (which is
   proprietary and uses dynamic spectrum allocation across Ku/Ka/W bands).

6. ALL ORBIT TYPES: The free-space path loss, atmospheric attenuation, and
   rain fade are computed dynamically by the link budget model using ITU-R
   P.618 and ITU-R P.676 propagation models.  They are NOT hardcoded in the
   orbit defaults.
